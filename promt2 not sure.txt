[BUG FIX] Playlist Screen: D-Pad Focus Traversal for Update & Delete Buttons

1. TARGET COMPONENTS
   ├── AREA    : Playlist Row Item Template.
   └── BUTTONS : "Update/Edit" Button and "Delete" Button icons.

2. PROBLEM DESCRIPTION
   ├── ISSUE  : When navigating the playlist using a TV Remote D-Pad, focus gets stuck on the main card or text. The user cannot move focus horizontally onto the Update or Delete buttons.
   └── DEFECT : These actions are currently only triggerable via touch pointer, locking out non-touch TV users.

3. FIX REQUIREMENTS
   ├── ACTION_1 : Explicitly mark both Update and Delete buttons as focusable and clickable (`focusable="true"`, `focusableInTouchMode="true"`, `clickable="true"`).
   ├── ACTION_2 : Set up proper spatial navigation routing. Pressing [Arrow RIGHT] from the playlist title/card should shift focus smoothly to the [Update Button], and another [Arrow RIGHT] should move it to the [Delete Button].
   └── ACTION_3 : Ensure a clear visual focus highlight (toggled background color or stroke tint) triggers when either button is active under D-pad selection.
